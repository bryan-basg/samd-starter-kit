# Roadmap

> The canonical, always-up-to-date roadmap lives at the repo root in
> **[`ROADMAP.md`](https://github.com/bryan-basg/samd-starter-kit/blob/main/ROADMAP.md)**.
> This page is a short mirror so the roadmap is reachable from the docs site.

The kit is a **process scaffold**, not regulatory advice and not a guarantee of
certification. Horizons are intentionally soft ("next", "exploring") — this is an
open-source side project, so we don't commit to calendar dates.

Want to shape it? Open a
[feature request](https://github.com/bryan-basg/samd-starter-kit/issues/new/choose),
grab a good first issue, or start a
[Discussion](https://github.com/bryan-basg/samd-starter-kit/discussions).

## Already shipped

- **Team of 10 agents** with the SaMD ruleset pre-loaded (`.claude/agents/`).
- **Agent ruleset** (`CLAUDE.md`) — Rule 0 and the multi-agent anti-drift protocol.
- **11 working CI/CD workflows** (`.github/workflows/`).
- **40+ document Design History File** (`docs/`).
- **Runnable skeleton** (`app/` · `frontend/` · `tests/`).
- **Worked example** (`examples/auralog/`) — a fictional Class B device.
- **Bilingual onboarding** (English / Spanish) with a pre-certification clinical glossary.

For the full, current list (in progress / next / exploring), open the canonical
[**`ROADMAP.md`**](https://github.com/bryan-basg/samd-starter-kit/blob/main/ROADMAP.md).
